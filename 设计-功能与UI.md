# 乐龄守护 · 功能与 UI 设计稿 (v1)

> 本文件只定义**有哪些功能、长什么样、怎么走**，不讨论技术实现。实现按 [PLAN.md](PLAN.md) 逐步填充。
> 选题背景见 [方案-适老健康自查App.md](方案-适老健康自查App.md)。

## 已拍板的产品基调

- **单人使用**：一部手机就一个老人，无多档案切换，UI 最简。
- **加 SOS 亮点**：全局「一键呼叫子女 / 紧急求助」，呼应"跌倒"主题。
- **只用系统分享**：发给子女走 PDF/图片 + 微信/短信，**无账号、无登录、无云端**。
- 定位：**健康筛查与趋势提示，非医疗诊断**。

## 一、信息架构（底部 4 个大 Tab + 全局 SOS）

```
┌──────────────────────────────────────────────┐
│   首页        自查        趋势        我的        │
│  house      heart      chart      person       │
└──────────────────────────────────────────────┘
        全局可达：🆘 紧急呼叫（见下）
```

### 🏠 首页（每天打开第一眼）
- 大字问候 + 日期
- AI 一句话健康提示（今天感觉如何 / 该注意什么）
- **超大主按钮：「开始今日自查」**
- 「最近一次」结果卡：心率 / 呼吸 / 跌倒风险，各带 🟢🟡🔴 等级
- 今日提醒：该测量了 / 该吃药了
- 顶部常驻 **🆘 紧急呼叫** 按钮（红色、最显眼）

### 💗 自查（核心）
- 两张大入口卡：
  - **跌倒风险评估**（⭐️ 星功能）
  - **心率 / 呼吸自查**
- 「一键全部测」：连续做完两项
- 最近几次记录的简单列表

### 📈 趋势
- 顶部切换：心率 / 呼吸 / 跌倒风险评分
- 大号粗线折线图
- 日历：哪天测了、当天结果颜色
- 一句话「本周 vs 上周」对比
- 「生成本周健康报告」入口

### 👤 我的
- 基本信息：年龄 / 性别（评估基准用）
- **紧急联系人**（SOS 拨打对象，可多个）
- 提醒设置：测量提醒、用药提醒
- 导出 / 发给子女（PDF / CSV / 分享）
- 关于 + 免责声明

## 二、核心流程（UI 走法）

### ① ⭐️ 跌倒风险评估
```
说明页（图示 + 语音："把手机靠墙立稳，露出全身"）
  → 取景对位（人形框引导："再往后站一点"）
  → 准备倒计时 3 · 2 · 1
  → 测试中（30 秒）：大数字显示已坐站 N 次 + 剩余秒数 + 语音报数
  → 结果页：次数 / 用时 / 🟢🟡🔴 风险等级 + 一句解读
            [保存] [发子女]
            ▸ 若结果为🔴偏高，额外提示"建议告知家人"并给 SOS 快捷入口
```

### ② 心率 / 呼吸自查
```
说明页（语音："保持不动约 30 秒"）
  → 测量中：大进度环 + 心跳动画 + 实时提示
  → 结果页：❤️ 心率 bpm + 🫁 呼吸 次/分 + 等级 + "比上次 ↑/↓"
            [保存] [发子女]
```

### ③ 一键全部测
```
自查页「一键全部测」→ 依次跑 ① 跌倒风险 → ② 心率呼吸 → 汇总结果页
```

### ④ AI 健康周报 + 发子女
```
趋势/我的 "生成本周报告"
  → AI 生成：一周摘要 + 异常提醒 + 暖心建议（大白话）
  → PDF 预览
  → 系统分享 → 微信 / 短信发给子女
```

### ⑤ 🆘 紧急呼叫 / SOS（亮点）
```
首页常驻红色「紧急呼叫」按钮（全局可达）
  → 点按 → 大确认页（防误触，3 秒倒计时可取消）
  → 确认 → 直接拨打紧急联系人电话
            （可选：同时发一条带当前状态/位置的短信）
  ▸ 紧急联系人在「我的」里预先设置
```

## 三、贯穿全局的适老规范

- **字大**：正文 ≥ 21pt，关键数字超大；跟随系统动态字体继续放大。
- **对比强**：文字/背景高对比，弱视也看得清。
- **语音**：每个自查流程全程语音引导 + 语音报结果。
- **颜色表意统一**：🟢 良好 / 🟡 注意 / 🔴 偏高·风险，全 app 一致。
- **大触控目标**：主按钮高 ≥ 64pt。
- **流程极简**：每个任务 ≤ 4 步，随时可"重测"。
- **防误触**：SOS、删除等关键操作带二次确认。
