## 〇、这次为什么改（一句话）

指导教授（浙大设计学院 × 苹果）点了三件事，归结成一句：

> **护城河不是"测得准"，是"老人愿意用、用了不难受"。**

- 技术门槛在降 → 评委（设计 + 苹果）不为算法鼓掌，为"有尊严的交互"鼓掌。
- 健康测试让老人焦虑 → 现在这版是"给子女监控父母的医疗工具"，要改成"让老人觉得被关心的东西"。
- 让老人做蹲起测跌倒太抽象/荒谬 → 用**椅子坐站**（本就是临床标准量表），且**别叫"测试"**。

---

## 一、v2 产品定位

> 一部旧手机，用**轻松、没有压力**的方式陪老人活动活动、感受一下身体状态；
> 数据在后台默默记录，**只有发给子女时才变成一份科学的健康报告**。
> 老人端：像陪伴，不像体检。**零硬件、全本地、不上云。**

**一句话卖点（对评委）**：同一个动作，**老人感到的是陪伴，子女收到的是安心**。

---

## 二、核心设计原则（v2 新增/强化）

1. **一个 App，两种语气（本版骨架）**
   - **老人端**：轻松、口语、鼓励，**不打分、不出现红色"风险"、不叫"测试/评估/自查"**。
   - **子女端 = 导出的 PDF/分享内容**：科学、严肃、有数据、有分级、有异常提醒。
   - 落地：同一份数据，两套"呈现层"。老人看 `warmView`，PDF 走 `clinicalReport`。

2. **零失败感（不是游戏化打卡）**
   - 不计分、不连续打卡、不勋章、不排名、没有"失败/不合格"状态。
   - 允许随时停下、随时重来，永远是"没关系，慢慢来"。
   - 反馈只有两种：**温暖的陪伴** + **小小的愉悦**（动效/音效/触感），不带竞争。

3. **奖励 = 情感连接，不是积分**
   - 每次活动做完，落点是"要不要把今天的好消息告诉闺女？" → 顺手接上"发子女"主线。

4. **适老无障碍是真功夫（苹果加分点）**
   - 特大字 + 全程支持 Dynamic Type 继续放大；关键流程 **VoiceOver / 语音全程可听**。
   - 触控目标 ≥ 64pt；触感反馈（Haptics）在关键节点给"确认感"。
   - **数据全本地、不上云**——强隐私叙事，写进 demo。

5. **降低"必须测"的压力**
   - 把"每天提醒我自查"这类**制造愧疚的提醒**弱化/改造成柔性邀请（"今天有空活动活动吗～"），可关。

## 四、信息架构（老人端）

**建议从 4 Tab 精简为 3 Tab**（少即是易），去掉老人看不懂也徒增焦虑的"趋势折线图"，趋势只进 PDF/子女端。

```
┌───────────────────────────────────────┐
│    今天          回顾          我的       │
│   （首页）     （温和回看）   （设置/子女）  │
└───────────────────────────────────────┘
         全局可达：📞 联系家人（见 §7，改造 SOS）
```

- **今天**：问候 + 一句暖心话 + 今天想做点什么（腿脚操 / 静一静）+ 上次记录（温柔呈现）。
- **回顾**：**不放临床折线**。放"这几天你都挺好的"式的温和小结 + 一个简单的"哪天动过"日历（只有小太阳/小花，无颜色分级）。真实趋势图只在 PDF。
- **我的**：家人（紧急联系人 + 发报告对象）、柔性提醒、基本信息、关于/隐私。

> ✅ **已定 A**：老人端精简为 3 Tab；折线趋势图**彻底移出老人端，只进 PDF**。（折线本是给自己看的减肥软件遗留，老人看不懂、徒增焦虑。）

## 五、逐功能规划（老人看到 / 后台算 / 子女 PDF 看到）

每个功能都按**三层**写清楚，实现时照层写即可。

> ✅ **已定 E（贯穿全节）**：**老人端不显示任何临床数字与分级**（连心率 bpm 都不直接甩给老人）。老人只看到"大概的、积极鼓励的"定性反馈（如"心跳很平稳呢""腿脚挺利索"）。**所有精确数字、区间、🟢🟡🔴 分级一律进 PDF/子女端。** 数据不是不记，是"记在后台、说给孩子、不压老人"。

### 5.1 腿脚操（原"跌倒风险评估"，⭐️ 英雄功能候选）

- **动作**：坐在椅子上 → 起立 → 坐下，重复。临床依据 = **30 秒椅子坐站测试（CST）/ 五次坐站（FTSST）**，老年医学标准量表，安全、规范、有面子。
- **老人看到**：
  - 引导："咱们陪您活动活动腿脚，坐在椅子上站起来、坐下就行，慢慢来，不着急。"
  - 过程中**不显示"第几次/倒计时成绩"**。用温柔动效替代：每起立一次，画面上开一朵花 / 飞一只鸟 / 亮一盏灯（**只增不减，无失败**）。语音："真棒，再来一个～"
  - 结束："做完啦，腿脚挺利索的，歇会儿喝口水。"（**不报分数、不报等级**）
- **后台算**：起立次数、每次耗时/速度、姿态稳定度 → CST 指标 → 风险分级（good/caution/risk）。**只写库、只进 PDF，不给老人看。**
- **子女 PDF**：坐站 N 次、平均 X 秒/次、风险等级 🟢🟡🔴、"较上周变化"、异常提醒。
- **技术**：Vision `VNDetectHumanBodyPoseRequest` 检测髋/膝关节点，判坐↔站切换计数。**这是本项目最大的新技术块**，也是最值得作为"活的 demo"的一块。

### 5.2 静一静 / 听听心跳（原"心率/呼吸自查"）

- **老人看到**：不叫"测量"。像一次引导放松：一个随呼吸缓缓涨缩的圆圈"跟着它深呼吸"，手指盖住后摄（现有手指法）。结束："心跳很平稳呢，真好。"**不显示 % 进度作为压力**，或把进度做成柔和的"快好了"。
- **后台算**：心率 bpm、呼吸次/分（**现有 `PPGMeasurer` 引擎直接复用**）。
- **子女 PDF**：心率 72 bpm、呼吸 16 次/分、区间是否正常、趋势。
- **技术**：`PPGMeasurer.swift` / `PPGMath` **保留不动**，只重做外层 UI 与结果文案（去掉给老人的 🔴 分级）。

### 5.3 给闺女报个平安（原"AI 健康周报 + 发子女"，主线奖励）

- **老人看到**：一句话邀请"要不要把这几天的好消息告诉孩子？"→ 一键 → 生成 → 系统分享（微信/短信）。老人**不需要读懂报告内容**，就是"报平安"。
- **子女收到（PDF/图片）**：**科学严肃语气**——姓名、周期、心率/呼吸/坐站数据表、趋势图、🟢🟡🔴 分级、异常提醒、暖心但专业的建议、免责声明。
- **技术**：
  - AI 文案：迁移 DeepSeek 客户端，**两套 prompt**——①老人端一句暖心话（口语）②PDF 专业小结（严肃）。默认用最新 Claude/DeepSeek 模型。
  - PDF：新写 `ClinicalReport` 渲染 + 系统 `ShareLink`。

### 5.4 联系家人（原 SOS，改造，见 §7）

### 5.5 柔性提醒（原"提醒设置"）

- 把"每天提醒自查"改成**可关的柔性邀请**，措辞去愧疚化。用药提醒保留（这个是真需求，老人自己也想要）。
- 技术：本地通知 `UNUserNotificationCenter`。

## 六、视觉 / 交互规范修订

> ✅ **总基调（已定 D）：治愈系 · 简约。** 大量留白、素净配色、超大字，对齐苹果审美——**不做童趣卡通**。动效温柔克制（缓慢、柔和），像冥想/呼吸类 app 的气质，而非儿童游戏的花哨热闹。

- **色彩语义调整**：`good/caution/risk` 三色**保留定义**（PDF/子女端要用），但**老人端界面尽量不出现 risk 红作为"评分"**。老人端主色仍是青绿 = 安心；红色只留给"联系家人"按钮和 PDF。
- **动效/音效/触感**：关键节点（起立一次、测完、发送成功）给**正向**的动效 + 轻触感 + 温柔音效。**绝不用错误音/震动惩罚**。
- **字体**：沿用 `SeniorTheme` 语义字号；确保跟随系统 Dynamic Type 放大不破版。
- **语音**：腿脚操、静一静全程 `AVSpeechSynthesizer` 语音引导 + 报结果（暖）。

## 七、SOS 改造（重要设计修正）

**问题**：现在首页顶部一个巨大红色🆘，每天开机第一眼 = "你很脆弱、随时可能出事" → 和"无负担"直接冲突。

**改法（待拍板 B）**：
- 老人端首页**不再把红色 SOS 作为最抢眼元素**。改成常驻但不喧宾夺主的"📞 联系家人"（青绿/中性），点开里面既能"平常打个电话"也能"紧急求助"，把"紧急"藏进"联系家人"里，降低恐吓感。
- 真·紧急能力保留：二次确认 + 倒计时 + 拨号 `tel://`。
- 自动跌倒报警（陀螺仪）**降级为可选后台能力**，不在老人日常界面强调（前提"手机带身上"老人常不满足，别当主打）。

> ✅ **已定 B**：SOS 从"首页最大红按钮"降级为中性的"📞 联系家人"入口，紧急能力藏在里面，不在日常界面吓人。

## 八、英雄功能 & Demo 路径

- **demo 台上必须有一个是"活的"**，能在 **iPhone 13 mini**（A15，支持 Vision）上真点起来。
- **英雄功能建议：腿脚操（坐站，5.1）** —— 它同时满足：① 手表做不到的差异化 ② 教授认可的椅子版 ③ "同一动作，老人陪伴/子女安心"的双语气叙事最完整 ④ Vision 视觉效果在演示台上最抓眼。
- **保底 demo**：静一静（心率呼吸）引擎已现成，作为"至少这个稳跑"的兜底。
- **叙事高潮**：现场做完腿脚操 → 一键"报平安" → 手机上弹出给女儿的专业 PDF ——**当场把"陪伴→安心"的转换演给评委看。**

> ✅ **已定 C**：**两个功能都保留、都做**——"英雄功能"不是二选一，只是指"demo 台上重点演、开发上最早启动"的那一个。
> - **腿脚操（坐站，Vision）= demo 主角**：新技术、最抓眼、双语气叙事最完整。
> - **静一静（心率呼吸，PPG）= 已现成的稳定搭档**：引擎已实现，作为兜底 + 日常轻量项，天天能用。
> - 两者在"今天"页并列入口，也可串成"一起做一遍"。

## 十、建议实现顺序（真机以 iPhone 13 mini 为准）

1. **先真机验现成的**：心率呼吸（`PPGMeasurer`）在 13 mini 上编译通过 + 测得出数（确认底座活着）。
2. **英雄功能**：腿脚操 Vision 坐站——先跑通"检测坐↔站计数"，再叠温柔动效层。**这块最花时间，最早启动。**
3. **双语气落地**：结果三层（老人暖 / 后台算 / PDF 严肃）+ PDF 渲染 + 一键分享。
4. **AI 暖心话 + 专业小结**（双 prompt）。
5. **SOS 改造 + 柔性提醒 + 语音引导**。
6. **适老无障碍全局复核**（Dynamic Type / VoiceOver / 触感）+ demo 脚本。
