# 乐龄守护 · 养花「状态镜子」主页设计

> 本文记录主页核心情感设计：**一盆会反映身体状态的花**。
> 由蚂蚁森林式「栽培可视化」讨论演化而来，最终定为「**状态镜子**」而非「养成系」。
> 配合 [设计-v2-轻松关怀版.md](设计-v2-轻松关怀版.md) 看；**取代 v2 里「今天」首页与「回顾」页的原设计**。
>
> ⚠️ 范围：**初赛只交概念稿**，本文是概念定稿，花的状态逻辑不要求初赛前做成功能（见 §5.1）。

---

## 一、一句话概念

> 主页正中一盆**鲜艳卡通的花**。老人身体状态好，花就**摇曳、阳光充足、精神**；
> 状态平平，花就**安安静静不摇**（**但绝不蔫、绝不枯**）。
> 花是老人和自己身体之间一个**温柔的中介**——不给数字、不打分，一眼就懂"今天挺好"。

---

## 二、为什么是「状态镜子」而不是「养成系」

聊的过程中出现过两套逻辑，最终定了后者：

| | 养成系（放弃） | ✅ 状态镜子（采用） |
|---|---|---|
| 花代表什么 | 你的努力/坚持累积 | 你**此刻的身体状态** |
| 怎么长 | 做活动→一点点长大→开满→换盆 | 不"长大"，而是**此刻的样子**在变 |
| 问题 | 和身体无关，显得空 | —— |
| 素材现实 | "种子→开花"生长动画**极少** | "摇摆/绽放"的花**遍地都是** |

> **关键洞察**：LottieFiles 上"摇摆绽放的花"多如牛毛、"种子长到开花"的完整生长动画少之又少。
> 素材市场恰好在替我们投票——难找的正是我们不要的（养成），好找的正是我们要的（状态镜子）。

---

## 三、铁律（不可动摇，护住"无负担"立身之本）

1. **只有积极状态，绝不负面。** 花只在"**精神**"和"**安静歇息**"之间变，**永远不蔫、不枯、不生病**。"不摇摆" = 安静歇息（还是一朵好花），不是奄奄一息。
2. **背景也不许阴沉。** 好 = 大晴天/阳光；歇息 = 柔和晨光/一点点云。**绝不暴雨阴天**。
3. **实话只对子女说。** 真正"身体不太好"的信号，进给子女的**严肃健康报告**（数字 + 🟢🟡🔴）。老人端那盆花**永远是好的**。—— 即我们定好的「一个 App 两种语气」。
4. 依据：受访百岁老人的心态哲学——**心态最重要**。app 的职责是替老人守住"没压力"的好心态，而不是把身体的坏消息天天糊在她脸上。

## 四、主页线框（花 = 状态镜子）

```
┌──────────────────────────────┐
│   ☀️ (阳光 / 天空背景，随状态变)  │  背景素材：待找（不 AI 画）
│                              │
│         🌸                   │  ← 那盆花，主角、唯一亮色焦点
│      精神 → 摇曳 + 阳光足       │     好：摇曳、光足、可加小亮点
│      歇息 → 安静不摇(不蔫)      │     平：静、柔光，但仍是好花
│                              │
│   "今天花儿精神着呢～"          │  ← 一句暖话，永远往好里说
│                              │
│ ┌──────────────────────────┐ │
│ │     看看今天怎么样  ▶       │ │  ← 一键（下方大按钮，见 §5.3/5.4）
│ └──────────────────────────┘ │
├──────────────────────────────┤
│    ☀️今天    🌱回顾    😊我的   │  Tab 不变
└──────────────────────────────┘
```

- **美术风格**：绿枝干 + 粉/黄花瓣、**鲜艳卡通**；**界面本身保持简约**（留白、大字、素净）。
  = "简约打底 + 一朵鲜艳卡通花点睛"（对 v2 决策 D「治愈系简约」的微调，非推翻）。
- 右上角仍可留「📞家人」入口（见 v2）。

## 五、交互决策（本轮四问的回答，已定）

### 5.1 花的状态反应逻辑 —— **初赛先放一放**
初赛只交**概念稿**，花随状态变化的映射**不要求做成功能**。初赛阶段：主页摆出这盆花 + 好看的背景 + 一键入口，把**概念表达清楚**即可（可先固定为"精神"态做展示）。功能化留到决赛/后续。

### 5.2 没测时 / 新的一天 —— **延续昨天的状态**
还没测、或新的一天还没测时，花**延续上一次的状态**显示。
> 注：这是"想当然"的占位假设，是否科学待后续再斟酌（比如新的一天要不要回中性"等你来"）。概念稿阶段够用。

### 5.3 「一键」测什么 —— **全部做一遍**
下方大按钮 = **一次把所有项目都做一遍**（腿脚操 + 静一静，即原 v2 的"一起做一遍"），做完统一反映到花上。

### 5.4 按钮措辞 —— **日常化，淡化"测试"**
不叫"测试/评估"。候选：**「看看今天怎么样」**（首选）/「今天来一下」/「陪它一会儿」。最终由用户定。

## 六、回顾页 = 花的历史（累积感放这里）

"跨天连起来/累积"的需求**不靠单株花生长**来满足，而是放进「回顾」页：
- 记录**这些天每天那朵花的样子**（哪天精神、哪天安静），一排小花铺开。
- **取代** v2 回顾页原来的"发芽日历"——比日历有温度。
- 于是分工清爽：**首页 = 此刻的花；回顾 = 这些天的花**。

---

## 七、美术 / 素材路线（人做，不 AI 画）

- **主路线：Lottie**（[airbnb/lottie-spm](https://github.com/airbnb/lottie-spm)，Apache-2.0，SPM 一键装）。
  - 花：挑一朵讨喜的**卡通摇摆/绽放**款（这种管够）。用户自己审美挑选。
  - "精神 vs 歇息"两档：起步用**一个素材 + 代码调"精神头"**（饱和度/摇摆速度/微光），不必找稀有的多状态素材。
  - 天气/阳光背景：**用户自己找**，不 AI 生成。
  - ⚠️ 每个 Lottie 文件**授权不同**，选定后逐个确认可免费用于比赛作品。
- **备选：Rive**（[rive-ios](https://github.com/rive-app/rive-ios)，MIT）。若以后想要"一朵花、一个状态输入、多状态平滑切换"，Rive 更合适（状态机天生适配），但需在编辑器里做/改文件。

## 八、（未来做功能时）状态从哪来

- 花的"精神 / 歇息"两档，由**最近一次测量结果**映射：心率 / 呼吸 / 腿脚（坐站）在正常区间 → 精神；偏离 → 歇息（**但仍是好花**）。
- 真实的偏离程度、分级 → 只进子女报告，不反映为"蔫"。
- 存储：复用 [LeLingStore](LeLing/Storage/LeLingStore.swift) 的记录，取最新一条判档。
