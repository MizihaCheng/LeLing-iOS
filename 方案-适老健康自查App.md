# 适老健康自查 App 方案

> 暂定名：**安康** / **乐龄守护**（待定）
> 用途：参加 Apple 移动应用创新赛（MACC）的选题方案，供组内讨论拍板。
> 背景：原 FitLog-iOS 是安卓健身 app 的移植（复刻），创新性不足；本方案将其转型为「适老健康自查」原创产品，**大量复用已有代码**，而非从零开始。

---

## 一、一句话定位

> 一部旧手机、对着摄像头，就能给老人做**基础健康自查**和**跌倒风险评估**，结果自动生成 AI 健康周报、一键发给子女。**零额外硬件，专为老人设计。**

---

## 二、要解决的真问题（pitch 故事线）

1. **痛点**：测心率、评估健康，现在基本得买上千块的智能手表——但大量老人**不戴、不会用、嫌麻烦、忘充电**，等于被挡在门外。
2. **市场空白**：国外人脸测生命体征（Anura / Binah.ai）只做 B2B、不做中国消费端；国内这块几乎全是硬件（华为 WATCH D2 / Apple Watch / 小米 / SKG / 毫米波雷达）。**没有一个纯手机、适老、带跌倒风险评估的消费级 app。**
3. **致命点**：跌倒是中国老人致残 / 致死的头号原因，而**智能手表恰恰测不好跌倒风险**（测不了起坐姿态 / 步态质量）。
4. **我们的解法**：用人人都有的旧手机摄像头，做手表做不到的事，且零花费、极致适老。→ 普惠 + 无障碍，正中 MACC 命题。

---

## 三、功能范围（关键：砍出一个能做精的 MVP）

### 核心三件（差异化、必做）

| 功能 | 技术 | 说明 |
|---|---|---|
| **① 跌倒风险评估**（主打星功能） | Apple Vision 人体姿态估计 | 摄像头自动跑临床「30 秒坐站测试」：数起坐次数 + 算速度 → 出风险等级。**手表做不了，临床有依据，演示稳。** |
| **② 一键健康自查** | 人脸 rPPG（前摄拍脸约 30s） | 测**心率 + 呼吸频率**，全程语音 / 大字引导。HRV 只标"趋势参考"，不号称医疗级。 |
| **③ AI 健康周报 + 发子女** | DeepSeek + 导出 / 分享 | 把趋势讲成人话、给异常提醒，一键 PDF / 微信发给在外地的子女。 |

### 支撑功能（直接复用 FitLog 代码）

- **健康记录 + 趋势图 + 日历**（FitLog 的 Store / Swift Charts / CalendarView 几乎照搬）
- **适老化外壳**：超大字、高对比、语音引导、极简流程
- **测量 / 用药提醒**（本地通知，简单）

### 明确砍掉（并向评委解释为什么）

血压、整夜睡眠监测、血氧、医疗级 HRV ——**纯手机做不准，硬塞 = demo 翻车**。
全产品定位「**健康筛查与趋势提示，非医疗诊断**」，附免责声明（沿用 HRR 功能已有的声明经验）。

---

## 四、技术复用 vs 新学（让组员看清工作量）

### ✅ 直接复用 FitLog-iOS（约省一半工作量）

- 本地 JSON 存储（FitStore 模式）
- 趋势图表（TrendsView / Swift Charts）
- 日历（CalendarView）
- AI 客户端 + 总结（DeepSeekClient + TodaySummary 模式）
- PDF / CSV 导出（PDFReport / CSVExport）
- 主题配色、Tab 架构、表单组件（FitColors / Components）
- **连之前写的摄像头 PPG 信号处理（HeartRateMeasurer）都不浪费**——把"手指通道均值"换成"人脸额头 / 脸颊 ROI"，去趋势 + 带通滤波 + FFT 那套 DSP 直接迁移。

### 🆕 需要新学的（两块）

1. **Apple Vision 姿态估计**（`VNDetectHumanBodyPoseRequest`）——本地、免费、不需联网。用于坐站计数和姿态分析。**新技术栈，但有官方文档和 GitHub 例子可抄。**
2. **人脸 ROI 定位**（Vision 人脸特征点找额头 / 脸颊）+ rPPG 取信号。

## 五、分步开发计划（延续"小步 push、Mac 真机验"工作流）

> 摄像头 + Vision 必须真机验证（模拟器没相机）。每步 push 后在 Mac `git pull` + Xcode ⌘R。

- **第 0 步｜立项**：决定用新仓库还是改 FitLog（见第六节）。搭好适老化 Tab 外壳，移植 Store / 图表 / 日历底座。
- **第 1 步｜数据模型 + 记录 / 趋势**：定义生命体征记录、评估记录模型；趋势页跑通（复用为主）。
- **第 2 步｜跌倒风险评估（先做星功能）**：Vision 姿态 → 坐站测试自动计数计时 → 风险等级。**优先做它**，因为最新、最关键，早点验证新技术栈可行性。
- **第 3 步｜人脸 rPPG 自查**：迁移 HeartRateMeasurer 到人脸 ROI，出心率 + 呼吸频率，语音引导。
- **第 4 步｜AI 健康周报 + 分享子女**：迁移 DeepSeek 总结，改 prompt 为"老人健康解读 + 异常提醒"，PDF / 分享。
- **第 5 步｜适老打磨 + demo**：大字 / 语音 / 提醒，写好比赛演示脚本。

## 六、风险 & 待拍板的决策

1. **仓库选择（重要）**：建议**开新仓库、复制可用代码重新提交**。因为 FitLog-iOS 的 git 历史明显是"安卓移植"，评委若看历史会坐实"复刻"印象；新仓库给一个干净的产品故事。代价是要把复用文件拷过去。→ **待定：新仓库 or 沿用 FitLog-iOS？**
2. **准确度边界**：全程"筛查 / 参考、非诊断"，不与 Anura / Binah 比精度。
3. **子女远程**：真·云端账号同步太重，MVP 先用**系统分享 / PDF**发给子女（微信 / 短信），账号体系以后再说。
4. **团队技能**：Vision 是新栈，需要组里有人能一起啃。

- 有**手表给不了的硬差异化**（跌倒风险姿态评估）。
